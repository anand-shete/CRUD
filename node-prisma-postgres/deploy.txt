// Creating AWS RDS database (maxx free tier)
Go to AWS RDS service 
    Create a database > Standard Create 
For Engine Options, choose
    PostgreSQL
For Templates, select 
    Free Tier Template
Availability and durability, choose
    Single-AZ DB instance deployment
Configure Settings
    Enter whatever DB instance identifier > Choose Proper Credentials management
Instance configuration
    Uncheck Enable storage autoscaling
Connectivity
    Compute resource > Connect to an EC2 compute resource.
    Above step will automatically configure appropriate VPC configs as that of the EC2 instance.
    Virtual private cloud (VPC) > Auto-configured
    Public access > "No" as a security practice.
    (optional) VPC security group (firewall) > Create new > Give it a name
    RDS Proxy > Uncheck Create an RDS Proxy
    (Optional) choose database port in Additional configuration.
Database authentication
    I choosed Password authentication as of this tutorial
Monitoring
    Keep it as is.
Additional configuration
    Backup > Uncheck Enable automated backup option
    Keep Encrption enabled
Finally, click Create database



// Connecting to AWS RDS database
Basically, if your RDS is publicly accessible and security group has an inbound rule of 
    Type: PostgreSQL
    Source: MyIP (best security practice)
Then, you should be able to connect to your AWS RDS instance from your laptop using psql. This is bad security practice.
A better way is to connect your RDS database from the EC2 instance, then use port forwarding using your ssh credentials.
    ssh -i ~/.aws/production-instance.pem -N -L 5436:<rds-endpoint>:5432 ubuntu@<ec2-public-ip>
Now, you have setup SSH tunnelling, connect to your AWS RDS through EC2 instance tunnel running at localhost 
    psql -U anand_shete -h localhost -p 5436 -d postgres
Note that password you enter is of your AWS RDS instance and not the local user. 
You may need to modify pg_hba.conf slightly and change the listen_addresses inside postgresql.conf even with SSH tunnelling.
DATABASE_URL="postgresql://[USERNAME]:[PASSWORD]@[HOST]:5432/[DATABASE_NAME]"

    npm install

    npm run build
Run prisma migrations in production
